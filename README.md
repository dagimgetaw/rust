## Rust 101

Learning Rust step by step through hands-on exercises, small projects, and real-world experiments.

### Why leaning rust

1. Fastest language
   Rust is a high-performance systems language comparable to C/C++. - Performance depends on how you write code - Not always “faster than everything” - But very close to C, sometimes equal

2. Rich type system
   Rust has no garbage collector and uses ownership + borrowing.
   - Prevents many runtime bugs
   - Forces you to model your system properly

3. No garbage collector
   Rust has no garbage collector and uses ownership + borrowing
   - No pauses (like garbage collactor languages)
   - Deterministic performance
   - Better for real-time systems

4. Useful compiler output
   Rust compiler is extremely strict and gives clear, actionable error messages
   - It teaches you while you code
   - Helps you fix bugs early
   - Prevents hidden issues

5. Memory safety
   Rust guarantees memory safety at compile time without runtime overhead
   - No null pointer crashes
   - No buffer overflows
   - No dangling pointers

6. Most beloved language (StackOverflow)
   Rust has been ranked among the most loved programming languages on Stack Overflow for years

7. Fastest adoption in various branches
   Rust is rapidly adopted in
   - Systems programming
   - Backend infrastructure
   - Blockchain
   - Embedded systems
   - Security-critical applications
